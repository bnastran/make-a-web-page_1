HTML

HTML is abbreviation for Hypertext Markup Language. HTML documents are hearth of the web. it consist from text content is what you see, and markup, what it looks like. Also can have references to other documents like images or video, and has links to other documents

HTML markup

HTML markup basic elements are Tags. It has following structure opening tag content closing tag Whole structure with opening tag, content and closing tag is ELEMENT.

Tree like structure

HTML element can have new elements inside of element, we are talking that it is tree like structure. Like in a tree, big branch has smaller branches and small branch has even smaller.

Webpage is a collection of boxes

Webpage is made of many boxes. Big box can be cut to smaller boxes. Boxes can me moved on page. Code writers use class attributes to label boxes. Writing HTLM code we use tag DIV to define a box.

Lesson 3: Adding CSS Style to HTML Structure

CSS

CSS is code written to control the "style" of HTML elements. CSS stands for Cascading Style Sheets.

"Cascading" means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements.

Adding CSS in web page

There are three metods we can use:

Method 1: write CSS in the head of your HTML - This method is good for very small projects

Method 2: Link your HTML to a separate CSS file - it lets you stay more organized when working on larger projects

Method 3: Write your style inline with your HTML - This is generally a very bad idea because it leads to lots of repeated code. When we decide to do small style change on site, and if change is on multipole location we have to make changes everywhere and there is increased risk to hit the problems.

Avoiding Repetition

Avoiding repetition is important principle of programming. Repetitions in programming could lead to two problems: errors and no consistency.

The programmer has to do same procedure over and over and if they don’t avoid repetition and in that case the risk of error increase. In his work programmer has to copy and past style attributes to many HTML elements. In case programmer decides to chance some attribute, it can lead to errors when they don't avoid repetition. If he doesn't avoid repetition, he is forced to do same changes everywhere the repeated code appears. Whole code can be very long and there is high probability that there will occur errors and mistakes. So avoiding repetition means also avoiding errors.

Avoiding repetition also means consistency. Programmer could assign for each »box« a div class and specify the style for specific class in CSS. Now the programmer has chance to change style of selected divs very simple, fast and without risk of error. All boxes with same div class will look exactly the same. They have same style which is specified in CSS.

Box Sizing and Positioning

SIZE

Each box has 4 components. Because there are so many components to each box, it can often be hard to get the size of a box just right. Another issue is that each browser word differentely and can display code differnt.

There are 2 techniques we can use:
Set sizes in terms of percentages rather than pixels.
Set the box-sizing attribute to border-box for every element.

POSITION

Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.

Adding the rule display: flex;to the appropriate CSS will override this behavior and let divs appear next to each other.
